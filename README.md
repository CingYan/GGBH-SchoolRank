# 🏯 宗門菁英整編 — 自動職位排序 MOD

鬼谷八荒宗門職位自動整編 MOD。

玩家不必是宗主；只要玩家目前有宗門，MOD 會在進入世界後與每個遊戲月份自動掃描同宗門 NPC，排除玩家本人，依照境界/等級、是否天驕與戰力候選排序，重新分配宗門職位。

## ✨ 功能目標

- 🧭 玩家是宗主、長老、真傳、內門、外門都可觸發，不限宗主
- 🚫 排除玩家，只調整 NPC 職位
- 🥇 依境界/等級優先排序
- 🌟 天驕作為同境界排序加權
- 🏯 職位排序：大長老 → 長老 → 真傳弟子 → 內門弟子 → 外門弟子
- 🕒 每個遊戲月份只自動整編一次，避免反覆洗職位

## 📌 初版配額

| 職位 | 配額 |
|------|------|
| 大長老 | 1 |
| 長老 | 3 |
| 真傳弟子 | 10 |
| 內門弟子 | 30 |
| 外門弟子 | 其餘 NPC |

> 配額先採保守預設，後續可依遊戲實測結果調整。

## 🔧 模組代碼

- 代碼命名空間 / AssemblyName：`MOD_SNs4Ii`

這是 v1 初版 skeleton：

- 已建立完整 MelonLoader MOD 架構
- 已實作同宗門 NPC 掃描、玩家排除、排序與職位寫入流程
- 職位 enum 會在啟動時輸出 `[ENUM] SchoolPostType=...` 供校準
- 若遊戲版本的 enum 名稱不同，log 會列出候選名稱，下一版可直接修正

## ⚠️ 測試重點

第一次測試請看 MelonLoader console：

- 是否出現 `[SchoolRank v1] === Init done ===`
- 是否印出 `[ENUM] SchoolPostType=...`
- 是否在進世界後出現 `[RANK] done ...`
- `[TOP]` 前幾名排序是否合理
- 宗門職位是否真的更新

如果 `[SET] enum candidates not found`，代表職位 enum 名稱和預估不同，把那段 log 貼回來即可修正。
